# TableTheory Evidence Plan (Draft)

This document defines where “evidence” for TableTheory’s planning rubric is produced and how to regenerate it.

Evidence should be reproducible from a commit SHA and should not require hand-assembled screenshots where a command,
report, or CI artifact is possible.

## Evidence sources

### CI artifacts (preferred)

- **Unit coverage**: generated by `bash scripts/verify-coverage.sh` (uploads `coverage_lib.out`).
- **Lint**: `bash scripts/verify-lint.sh` output.
- **Security scans**:
  - `bash scripts/sec-gosec.sh` (console + optional SARIF)
  - `bash scripts/sec-dependency-scans.sh` (console output; includes govulncheck + npm audit)

### Deterministic in-repo artifacts

- Controls matrix: `docs/development/planning/theorydb-controls-matrix.md`
- Versioned rubric: `docs/development/planning/theorydb-10of10-rubric.md`
- Roadmap mapped to rubric IDs: `docs/development/planning/theorydb-10of10-roadmap.md`
- Branch/release policy: `docs/development/planning/theorydb-branch-release-policy.md`
- Encryption tag roadmap: `docs/development/planning/theorydb-encryption-tag-roadmap.md`
- Maintainability roadmap: `docs/development/planning/theorydb-maintainability-roadmap.md`
- Threat model: `docs/development/planning/theorydb-threat-model.md`
